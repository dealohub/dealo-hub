## Why This Expansion

الـ 10-category plan الأصلي كان ناقص بشكل ضخم. تحليل الـ screenshots من Q84sale + Dubizzle أوضح:
- Kuwait user يتوقع **full-spectrum marketplace** في مكان واحد
- استبعاد cars/real estate/jobs = استبعاد 70%+ من الطلب
- "Boutique niche" play لا يبني network effects

**قرار جديد:** Dealo Hub = **منصة شاملة world-class** — مش niche.

## Schema Implications

The base `categories` table already supports this via `parent_id` hierarchy. Additional work needed:

### Phase 2 schema additions:
```sql
-- Category-specific fields (JSONB for flexibility)
ALTER TABLE listings ADD COLUMN category_fields JSONB DEFAULT '{}'::jsonb;

-- Validated per category via app-layer Zod schemas:
-- automotive/used-cars:     { make, model, year, mileage_km, transmission, fuel_type, vin, accident_history }
-- real-estate/rent:         { property_type, bedrooms, bathrooms, area_sqm, furnished, utilities_included }
-- watercraft:               { type, length_ft, engine_hp, hull_material, year }
-- motorcycles:              { make, model, year, cc, mileage_km }
-- jobs/vacancies:           { salary_range, contract_type, industry, experience_level }
-- services:                 { service_type, availability, service_area_areas }
```

### Phase 2 search indexes:
Custom GIN indexes on frequently-filtered `category_fields` keys per category.

### Phase 2 vertical-specific forms:
Each big vertical gets a tailored listing creation flow. Not a single generic form anymore.
