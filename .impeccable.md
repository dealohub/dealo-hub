# Design Context — Dealo Hub

> Gathered by impeccable skill on 2026-04-23 from codebase discovery (README, categories taxonomy, brand color usage, existing shadcn token system). Refine via `/impeccable teach` if the product direction shifts.

## Users

**Who**: Kuwaiti and GCC consumers browsing a C2C marketplace for cars (مركبات), electronics (إلكترونيات), properties (عقارات), home (منزل), services (خدمات), and jobs (وظائف). Bilingual — Arabic (RTL) default, English (LTR) secondary.

**Context**: Used across the day on mobile and desktop. Trust is the #1 concern — the tagline is *"AI-Protected, Human-Written Listings"*. Users commit to financial decisions (buying a 52,000 KWD Porsche, renting an apartment), so the interface must feel trustworthy on first glance.

**Jobs-to-be-done**: Find a vetted seller, verify the listing is real, negotiate, commit. Not impulse shopping — high-consideration purchases.

## Brand Personality

Three words: **Trustworthy · Regional · Confident**

- **Trustworthy** — AI verification, human curation, seller ratings, transaction protection
- **Regional** — Gulf-first (Kuwait launch, GCC expansion), Arabic-typographically native via Cairo font, KWD-first pricing
- **Confident** — strong brand red as the hero CTA color, direct copy ("بع الآن" / "ابدأ محادثة"), no timid greys

Anti-references: SOUQ's cluttered aggregation, Dubizzle's dated feel, generic OLX sameness.

## Aesthetic Direction

- **Theme**: Dual (dark + light). Light mode = daytime browsing + legal/document surfaces; dark mode = evening detail-page immersion (already executes well, the redesign keeps it untouched).
- **Typography**: Cairo (Arabic, whole RTL tree) · Geist Sans (Latin body) · Cal Sans (display accents). Established, don't touch.
- **Tinted surfaces**: Must read as neutral, not coloured. The marketplace's identity comes from category accents (red cars, amber motorcycles, sky watercraft, stone CMVs), not from global UI tinting.

## Color System (established)

- **Brand red**: `#e30613` → HSL `357 94% 46%`. Hot accent, used for primary CTAs, live-pulse dots, active-state underlines, category chips for cars/rides.
- **Ride category palette** (`src/lib/rides/queries.ts:168`):
  - Cars (default): `#ef4444` rose-red
  - Motorcycles: `#f59e0b` amber
  - Watercraft: `#0ea5e9` sky
  - CMVs/food-trucks: `#78716c` stone
- **Category vehicle swatches** (paint colors): broad warm+cool set across `src/lib/rides/color-swatches.ts` (white, pearl, black, silver, champagne, brown, red, blue, green, etc.) — these are PRODUCT colors, not UI colors, and must stay untinted on the card.
- **Primary token**: currently `oklch(0.58 0.19 274)` — purple-blue. **Wrong.** Never intentionally used; inherited from shadcn default. Should align with brand red.

## Design Principles

1. **Category accents own the color** — the global UI stays neutral so brand red + category tints pop. If the chrome is also tinted, the accents flatten.
2. **Tinted neutrals, not pure grey** — per impeccable: add a subtle hue nudge for cohesion, but under 0.01 chroma / 5% saturation. Invisible tint, maximum harmony with the brand red.
3. **Single-hue neutrals** — foreground and background share the same hue so `bg-foreground/[opacity]` mixes stay perceptually neutral, never shift toward a competing color. (The 2026-04-23 pink-tint bug was caused by hue 30 fg over hue 60 bg.)
4. **Warm stone is aspirational but high-risk** — Linear/Arc/Vercel pull it off with extreme precision. At our stage, a cool-neutral-with-whisper-of-brand-warmth is safer and the difference is imperceptible to end users.
5. **Dark mode is proven; don't regress it** — palette updates target `:root` only. `.dark` stays as-is.

## Accessibility

- WCAG AA minimum (4.5:1 text, 3:1 UI) — enforce on every token pair.
- `prefers-reduced-motion` already respected globally in `app/globals.css`.
- Bilingual RTL/LTR — all color decisions must work under `dir="rtl"` (mirror logic already handled at the layout/Tailwind-logical level; palette is agnostic).
